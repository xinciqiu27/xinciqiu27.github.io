## 二、论文内容
### 1. 主要思想

SLGGCL 针对图表示学习中的一个老问题：很多图对比学习方法只强调局部或全局的一种视角，没有同时刻画多尺度结构。

因此，论文提出同时进行 **局部视图对比** 和 **全局视图对比**，让节点分类模型既能保留局部邻域细节，也能利用扩散图中的全局结构信息。

---

### 2. 解决的问题

作者认为现有图对比学习存在两类不足：
- 多尺度属性利用不足，导致深层结构信息没有被充分捕获。
- 图增强策略和对比样本设计往往没有把语义信息与多尺度结构结合起来。

所以 SLGGCL 选择同时构造局部和全局两类视图。

---

### 3. 方法框架

SLGGCL 的核心流程是：
- 用边扰动、属性掩码等方式构造局部子图视图。
- 用图扩散构造全局视图。
- 在局部和全局多尺度图之间进行对比学习，提升节点表示质量。

这个方法更像是把“多尺度增强 + 对比学习”打包在一起。

---

### 4. 我认为值得关注的细节

对 Web API / 推荐方向而言，这篇论文的直接任务不相同，但它对“多尺度图对比学习怎么做”很有参考价值，尤其适合当作方法背景。

---

## 三、实验
### 1. 数据集与设置

论文在多个真实数据集上做图节点分类实验，并与 10 个 SOTA 方法比较。

---

### 2. 实验结论

文中提到，SLGGCL 在 Cora 上相较最优基线带来约 1.01% 的准确率提升，整体优于十个强基线。

---

## 四、论文亮点

- 同时建模局部与全局对比视角，设计比较完整。
- 图扩散视图引入了更强的全局结构感知能力。
- 适合作为多尺度图对比学习的通用参考。

---

## 五、局限性与我的批判性思考

- 局部 / 全局视图构造策略较多，调参成本不低。
- 在推荐图上能否稳定迁移，还要看用户-项目异构结构的适配性。
- 多尺度对比收益在稀疏超大图上是否仍然明显，需要进一步验证。

---

## 六、论文总结（个人理解）

我觉得 SLGGCL 的主要价值是提供了一套比较规范的多尺度图对比学习视角。虽然它不直接做 Web API 推荐，但它对后续多关系图建模和视图增强设计有借鉴意义。
